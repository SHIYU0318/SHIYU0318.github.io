資料結構 Data Structure

發表於2022-12-18|更新於2022-12-21|程式筆記
|字數總計:1.4k|閱讀時長:4分鐘|閱讀量:

資料結構 Data Structure 是電腦中儲存、組織資料的方式 正確的資料結構選擇可以提高 演算法 的效率

演算法 的筆記請點擊 這裡

陣列 Array

陣列是一種 線性串列

  • 由第 0 項開始
  • 儲存相同類型資料
  • 使用一段連續的記憶體空間來存放
  • 須事先宣告固定記憶體空間 容易造成浪費
  • 讀取與修改串列的時間快速
  • 刪除或插入資料需移動大量資料

連結串列 Linked list

  • 記憶體位置不連續 隨機儲存
  • 不需事先定義連續記憶體空間
  • 利用每個node中的pointer指向下一個node
  • 以NULL來代表Linked list的終點
  • 刪除或插入資料方便
  • 查詢特定節點時 必須從頭開始走訪

以兩個class(類別)表示Linked list

若以C++的class(類別)來實作Linked list，可以利用兩個class
使得node的資料不會被任意更動(也就是封裝(Encapsulation)的概念)

堆疊 Stack

「Last-In-First-Out」 後進先出 的特性

常用於網頁 undo 回到前一頁 的功能

處理資料結構的功能

  • Push(data)：把資料放進Stack
  • Pop：把最上面的資料從Stack中移除
  • Top：回傳最上面的資料
  • IsEmpty：確認Stack裡是否有資料
  • getSize：回傳Stack裡的資料個數

Top/IsEmpty/getSize 皆不影響資料結構本身

可以分別使用 Array 與 Linked list 來實作 Stack

佇列 Queue

「First-In-First-Out」 先進先出 的特性

常用於先到先執行 需要排程的應用

處理資料結構的功能

  • Push(data)：把資料從Queue的後面放進Queue 並更新成新的 back
    • 在Queue中新增資料又稱為 enqueue
  • Pop：把front所指向的資料從Queue中移除 並更新 front
    • 從Queue刪除資料又稱為 dequeue
  • getFront：回傳front所指向的資料
  • getBack：回傳back所指向的資料
  • IsEmpty：確認Queue裡是否有資料
  • getSize：回傳Queue裡的資料個數

可以分別使用 Array 與 Linked list 來實作 Stack

集合 Set

不能重複

順序 沒有意義 的資料

運算型態

Python 中 集合 Set 可使用的 方法(method)

方法(method) 描述
add() 為集合添加元素
pop() 隨機移除元素
remove() 移除指定元素
clear() 移除集合中的所有元素
copy() 複製一个集合
union() 回傳兩集合的聯集
update() 為集合添加元素
difference() 回傳多個集合的差集
difference_update() 移除集合中的元素 該元素在指定的集合也存在
discard() 刪除集合中指定的元素
intersection() 回傳集合的交集
intersection_update() 回傳集合的交集
isdisjoint() 判斷兩集合是否包含相同的元素 如果没有 回傳 True 否則回傳 False。
issubset() 判斷指定集合是否為該方法参數集合的子集
issuperset() 判斷該方法的参數集合是否為指定集合的子集
symmetric_difference() 回傳兩集合中不重複的元素集合
symmetric_difference_update() 移除當前集合中在另外一个指定集合相同的元素 並將另外一个指定集合中不同的元素插入到當前集合中

映射 Map

在Python中又稱 Dict 字典 是以 鍵-值對(key-value) 存取的資料容器

樹 Tree

結構

  • 節點 Node
  • 根 root
  • 葉節點 leaf
  • 父 parent
  • 子 child
  • 兄弟 siblings

特性

  • 不包含迴路
  • 任意兩節點僅有唯一的一條路徑
  • 有 n 個節點 則會有 n - 1 條邊

堆積 Heap

最小堆積 Min Heap

母節點的值恆≤子節點的值

最大堆積 Max Heap

母節點的值恆≥子節點的值

操作 描述 時間複雜度
build 採用羅伯特·弗洛伊德提出的較快方式建立堆積 O(n)
insert 向堆積中插入一個新元素 O(log n)
update 將新元素提升使其符合堆積的性質 O(log n)
get 取得當前堆積頂元素的值 O(1)
delete 刪除堆積頂元素 O(log n)
heapify 使刪除堆積頂元素的堆積再次成為堆積 O(log n)

雜湊 Hash

常被用來加密存在資料庫中的密碼

雜湊函式把訊息或資料壓縮成摘要 使得資料量變小 將資料的格式固定下來

該函式將資料打亂混合 重新建立一個叫做雜湊值的指紋(通常用一個短的隨機字母和數字組成的字串來代表)

由於雜湊演算法所計算出來的 雜湊值(Hash Value) 具有 不可逆 的性質(無法逆向演算回原本的數值)

因此可有效的保護密碼

雜湊表(Hash table)：是根據 鍵(Key) 而直接查詢在記憶體儲存位置的資料結構也就是說 它通過計算出一個鍵值的函數 將所需查詢的數據 映射(Map) 到表中一個位置來讓人查詢這加快了查找速度 這個映射函數稱做 雜湊函數 存放記錄的數組稱做 雜湊表

常見的雜湊演算法

  • MD5
  • SHA
